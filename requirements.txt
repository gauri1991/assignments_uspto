# Runtime dependencies for the USPTO patent-assignment tool (mirrors pyproject.toml).
#
# One-command setup to RUN THE DESKTOP UI from this folder:
#     python -m venv .venv
#     .venv\Scripts\Activate.ps1          # Windows PowerShell  (Linux/Mac: source .venv/bin/activate)
#     pip install -r requirements.txt
#     python run_ui.py
#
# The trailing `-e .` installs THIS project (editable), which is what makes
# `import uspto_assignments_ui` resolve to .\src — required because the package lives under src/.
# (pyproject.toml stays the source of truth; run `pip install -e ".[dev]"` for the dev tools/stubs.)

# --- optional ML classify backend (opt in — NOT installed by default) ---
# The rule-based classifier is the DEFAULT and needs none of this; it runs compiler-free on any
# supported Python (3.12-3.14). classify falls back to rules (with a one-time note) if a step selects
# ML while the backend can't load.
#
# The ML backend needs the C extension `python-crfsuite`, whose newest release (0.9.12) has wheels
# only up to CPython 3.12 — NONE for 3.13 or 3.14. So a compiler-free ML install requires a
# Python 3.12 venv; on 3.13/3.14 pip would build python-crfsuite from source and need a C++ compiler
# (there is no pure-Python CRF backend). To enable ML on Python 3.12 (Windows shown; the pure-Python
# `metaphone` shim in classify.py removes probablepeople's other C dep, `doublemetaphone`):
#
#     py -3.12 -m venv .venv312
#     .venv312\Scripts\Activate.ps1                 # Linux/Mac: source .venv312/bin/activate
#     pip install -r requirements.txt               # the base tool
#     pip install python-crfsuite probableparsing Metaphone
#     pip install --no-deps probablepeople
#
# Smoke test (expect: individual):
#     python -c "from uspto_assignments import classify_name; print(classify_name('SMITH, JOHN A', method='probablepeople'))"

# --- desktop UI ---
PyQt6>=6.6
